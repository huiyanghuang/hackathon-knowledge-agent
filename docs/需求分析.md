# 需求分析

## 1. 项目背景

医学院校通常同时使用多本教材讲授相关学科知识（如解剖学、生理学、病理学等）。不同教材对同一知识点的表述深度、侧重点、术语用法各有差异，学生难以形成统一、完整的知识体系。

本项目旨在构建一个**学科知识整合智能体**，自动完成：

- 多教材知识点的结构化提取
- 跨教材知识点的去重与对齐
- 知识图谱的可视化呈现
- 基于整合知识库的智能问答与多轮对话

## 2. 功能需求

### 2.1 教材管理

| 功能 | 说明 |
|------|------|
| 批量上传 | 支持 PDF、DOCX 格式，可批量上传多本教材 |
| 自动解析 | 提取正文文本，按章节分块（每25页为一组） |
| 进度跟踪 | 实时显示解析、提取、索引各阶段进度 |
| 教材列表 | 展示已处理教材的基本统计信息 |

### 2.2 知识图谱构建

| 功能 | 说明 |
|------|------|
| 知识点提取 | 每个文本块调用 LLM，输出结构化节点（名称、定义、类别）和关系（边）|
| 跨教材对齐 | Embedding 相似度 + LLM 判定，自动合并跨书同义知识点 |
| 图谱可视化 | ECharts 力导向图，节点颜色区分教材来源，支持拖拽/缩放 |
| 整合报告 | 输出合并决策列表，说明每次合并的理由和置信度 |

### 2.3 智能问答（RAG）

| 功能 | 说明 |
|------|------|
| 混合检索 | Gemini Embedding 向量检索 + BM25 关键词检索，双路 rerank |
| 带引用回答 | 答案附带原文来源（教材名、章节、页码范围）|
| 教材过滤 | 可指定只在某几本教材范围内检索 |
| 上下文保持 | 多轮对话，保留历史 messages |

### 2.4 对话式整合调整

| 功能 | 说明 |
|------|------|
| 多轮对话 | 与 AI 教师自然语言交互，讨论知识点整合决策 |
| 调整建议 | AI 可建议修改合并策略，解释不同教材的表述差异 |

## 3. 非功能需求

| 需求 | 指标 |
|------|------|
| 响应速度 | RAG 问答 < 10s（不含首次冷启动嵌入） |
| 知识压缩比 | 整合后知识点总字数 ≤ 原始总字数的 30% |
| 准确性 | 同义知识点合并误报率 < 10%（LLM 二次验证保障）|
| 离线运行 | 除 Gemini API 外无外部依赖，可本地部署 |

## 4. 用户角色

- **学生**：上传教材、浏览知识图谱、提问
- **教师**：审查整合决策、通过对话调整合并策略

## 5. 系统边界

- 输入：PDF / DOCX 格式教材（中文为主）
- 输出：知识图谱（JSON + 可视化）、整合报告（Markdown）、问答结果（带引用）
- 外部依赖：Google Gemini API（LLM + Embedding）
