## 2. 数据流

### 2.1 教材处理流水线

```
PDF文件
  │
  ▼ parse_pdf()
Textbook（章节列表，每块≈25页）
  │
  ▼ extract_textbook_knowledge()   [LLM×N次]
nodes[], edges[]  →  保存到 data/processed/{tid}.json
  │
  ▼ index_textbook()               [Gemini Embedding]
VectorStore（vectors.npz + metadata.json）
```

### 2.2 跨教材对齐流水线

```
多本教材的 nodes[]
  │
  ▼ _embed() [Gemini gemini-embedding-001]
N×D 嵌入矩阵
  │
  ▼ 余弦相似度矩阵
  │
  ├─ score ≥ 0.88  →  直接合并
  ├─ 0.70 ≤ score < 0.88  →  LLM 二次判定
  └─ score < 0.70  →  保留独立节点
  │
  ▼ align_nodes()
merged_nodes[], decisions[]
```

### 2.3 RAG 问答流水线

```
用户问题
  │
  ├─▶ Gemini Embedding (RETRIEVAL_QUERY)
  │     └─▶ 向量检索 top-10
  │
  ├─▶ BM25 关键词检索 top-10
  │
  ▼ 混合 rerank（向量分 × 0.7 + BM25分 × 0.3）→ top-5
  │
  ▼ 拼接上下文 + 构造 Prompt
  │
  ▼ Gemini LLM 生成
  │
  ▼ 返回答案 + 引用列表
```

## 5. 关键技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| LLM | Gemini gemini-3.1-flash-lite | 速度快、成本低、中文优秀 |
| BM25 分词 | jieba | 中文分词支持 |
| Embedding | Gemini gemini-embedding-001 | 无需本地模型，768维，质量高 |
| 向量存储 | numpy（自实现） | 无 C++ 编译依赖，轻量可移植 |
| PDF解析 | pypdf 4.x | 纯 Python，跨平台 |
| Web框架 | FastAPI + uvicorn | 异步、自动文档、类型安全 |
| 前端 | React + Vite + ECharts | 快速开发，图谱渲染能力强 |

## 6. 数据模型

### KnowledgeNode
```python
class KnowledgeNode(BaseModel):
    id: str
    name: str           # 知识点名称
    definition: str     # 定义/描述
    category: str       # 类别：核心概念 / 定理 / 方法 / 现象 / 原理
    chapter: str        # 所在章节标题
    page: int           # 章节起始页
    textbook_id: str
    textbook_name: str
    frequency: int = 1  # 出现在几本教材中（合并后）
```

### KnowledgeEdge
```python
class KnowledgeEdge(BaseModel):
    source: str         # 节点ID
    target: str         # 节点ID
    relation_type: str  # 关系类型（包含/前提/并列等）
    description: str
```

### MergeDecision
```python
class MergeDecision(BaseModel):
    decision_id: str
    action: str         # "keep" | "merge"
    affected_nodes: list[str]
    result_node: str
    reason: str
    confidence: float
```
